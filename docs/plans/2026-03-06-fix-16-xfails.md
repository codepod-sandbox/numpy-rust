## Task A: clip casting=None TypeError

**Tests fixed:** `TestClip.test_simple_int32_inout[0]` (1 test)

**What's happening:** `a.clip(m, M, out=int32_array, casting=None)` should raise `TypeError` because:
- Input is `int32`, bounds are `float64`, so the clipped result dtype would be `float64`
- With `casting=None` → "same_kind" semantics, writing `float64` into `int32` out is invalid
- NumPy raises `TypeError: Cannot cast ufunc 'clip' output from dtype('float64') to dtype('int32') with casting rule 'same_kind'`

**Current behavior:** `casting=None` is silently treated as no casting check; the value is truncated to int32 without error.

**Files:**
- Modify: `crates/numpy-rust-python/src/py_array.rs` (the `fn clip` method)

**Step 1: Understand the current clip signature**

In `py_array.rs`, search for `fn clip`. The method:

2. Reads `a_min`, `a_max`, `out` from args
3. Clips the data and writes to `out`

**Step 2: Implement the casting check**

In `fn clip`, after parsing args, add a check: when `casting=None` and `out` is provided, determine if result dtype is compatible with `out.dtype` under "same_kind" semantics.

Same-kind rule: `float → int` = NOT same-kind. `int → float` = NOT same-kind. `float → float` = ok. `int → int` = ok (if not narrowing, but same-kind allows it).

Specifically: if the input array has integer dtype and min/max are float, the result dtype becomes float. Writing float to int out = not same-kind → TypeError.

# These are what type(np.dtype('float64')) returns in real NumPy.
class _DTypeMetaBase:

    type = None   # scalar type (e.g. float64, int32, ...)

**Step 4: Make `type(dtype_instance)` return the right DType class**

This requires making each dtype instance an instance of the corresponding DType class rather than the generic `dtype` class. The cleanest way: make the `dtype` factory function return an instance of the specific DType subclass.

Currently `dtype` is a Python class. Change it so that `dtype('float64')` returns an instance of `Float64DType` (which is itself an instance of `dtype` via inheritance). This requires making `Float64DType` inherit from `dtype` AND be the actual class of the returned object.

Alternative (simpler): Override `dtype.__class_getitem__` and `dtype.__class__` dynamically — but `type()` can't be overridden easily.

Simplest approach: Make the `dtype` constructor return a different type:

```python
class dtype:
    # existing implementation...

    def __new__(cls, spec, align=False, copy=False):
        # Determine the specific DType class
        ...
        dtype_class = _DTYPE_CLASS_MAP.get(normalized_name, cls)
        instance = object.__new__(dtype_class)
        return instance

    def __init__(self, spec, align=False, copy=False):
        # existing init code
        ...
```

But `_DTYPE_CLASS_MAP` needs `dtype` to be defined first (since DType classes inherit from `dtype`). This creates a circular dependency.

**Solution:** Define `_DTYPE_CLASS_MAP` AFTER all DType classes are created, and set it on the `dtype` class:

```python
class dtype:
    _dtype_class_map = {}  # filled in later

    def __new__(cls, spec, align=False, copy=False):
        ...
        target_cls = dtype._dtype_class_map.get(normalized, cls)
        if target_cls is not cls and cls is dtype:
            instance = object.__new__(target_cls)
            return instance
        return object.__new__(cls)

# ... all DType classes defined here inheriting from dtype ...

# Then at the end, fill in the map:
dtype._dtype_class_map = {
    'float64': Float64DType,
    'int32': Int32DType,
    # etc.
}
```

This way `type(np.dtype('float64'))` = `Float64DType`. ✓

**Step 8: Fix strides getter to return F-order strides**

In `fn strides` in `py_array.rs`, when `inner.is_f_order`, compute F-order strides:

```rust
fn strides(&self, vm: &VirtualMachine) -> PyObjectRef {
    let inner = self.data.read().unwrap();
    let shape = inner.shape();
    let itemsize = inner.dtype().itemsize();

    let strides: Vec<i64> = if inner.is_f_order {
        // F-order: first dim has stride itemsize, each dim i = stride[i-1] * shape[i-1]
        let mut s = vec![0i64; shape.len()];
        if !shape.is_empty() {
            s[0] = itemsize as i64;
            for i in 1..shape.len() {
                s[i] = s[i-1] * shape[i-1] as i64;
            }
        }
        s
    } else {
        // C-order: last dim has stride itemsize (existing code)
        let mut s = vec![0i64; shape.len()];
        if !shape.is_empty() {
            *s.last_mut().unwrap() = itemsize as i64;
            for i in (0..shape.len()-1).rev() {
                s[i] = s[i+1] * shape[i+1] as i64;
            }
        }
        s
    };

    // Return as Python tuple
    let items: Vec<PyObjectRef> = strides.iter().map(|&s| vm.ctx.new_int(s).into()).collect();
    vm.ctx.new_tuple(items).into()
}
```

class _StrBytesArray:
    """Array for str/bytes dtype that tracks proper strides."""
    def __init__(self, data, shape, dtype_str, fill_value=None):
        self._data = data  # flat list of strings/bytes
        self._shape = tuple(shape)
        self._dtype_str = dtype_str  # 'str' or 'bytes'
        self._fill = fill_value

    @property
    def shape(self):
        return self._shape

    @property
    def ndim(self):
        return len(self._shape)

    @property
    def dtype(self):
        return dtype('<U1' if self._dtype_str == 'str' else '|S1')

    @property
    def strides(self):
        """C-order strides based on shape and itemsize."""
        itemsize = 4 if self._dtype_str == 'str' else 1
        s = [0] * len(self._shape)
        if self._shape:
            s[-1] = itemsize
            for i in range(len(self._shape) - 2, -1, -1):
                s[i] = s[i+1] * self._shape[i+1]
        return tuple(s)

    @property
    def size(self):
        n = 1
        for d in self._shape:
            n *= d
        return n

    @property
    def flags(self):
        return type('flags', (), {'c_contiguous': True, 'f_contiguous': len(self._shape) <= 1})()

- New `ArrayData::Structured { fields: Vec<(String, NdArray)>, shape: Vec<usize> }`
- Field access via string key: `arr['column_name']`
- Required for DataFrame column storage

**View semantics** (4-6 weeks Rust work):
- `PyNdArray` needs: `data_buffer: Arc<RwLock<Vec<u8>>>`, `offset: usize`, `strides: Vec<isize>`
- Slices become views (same buffer, different offset/stride)
- `shares_memory()` compares buffer identity + byte range overlap
- Required for pandas `.values`, `.iloc`, `.loc` returning views

**0-d arrays** (1-2 weeks):
- Scalar inputs create 0-d arrays with shape `()`
- Required for numpy scalar operations used by pandas

These should be planned as a separate session focused on "pandas compatibility foundations."
